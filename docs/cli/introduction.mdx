## Overview

The SuperBox CLI is a Python (Click) tool for managing MCP servers end to end. Commands map directly to the backend implementation in `superbox.cli` (see repo `SuperBox-BE/src/superbox/cli`), so usage here matches the code.

## Install

```bash
pip install superbox
```

## Quick Start

<Steps>
  <Step title="Authenticate">
    ```bash
    superbox auth login
    ```
    Device flow for Google/GitHub is supported; email/password works too.
  </Step>
  <Step title="Init server config">
    ```bash
    superbox init
    ```
    Creates `superbox.json` with name, repo URL, entrypoint, license, etc.
  </Step>
  <Step title="Publish with scans">
    ```bash
    superbox push --name my-mcp
    ```
    Runs SonarCloud, Bandit, GitGuardian, tool discovery, then uploads to the registry (S3).
  </Step>
  <Step title="Configure client">
    ```bash
    superbox pull --name my-mcp --client cursor
    ```
    Clients supported: `vscode`, `cursor`, `windsurf`, `claude`, `chatgpt`.
  </Step>
  <Step title="Run & view logs">
    ```bash
    superbox run --name my-mcp
    superbox logs --name my-mcp --follow
    ```
  </Step>
</Steps>

## Commands (source-of-truth)

- `superbox init` — create `superbox.json` interactively.
- `superbox auth` — grouped commands:
  - `superbox auth register`
  - `superbox auth login [--provider email|google|github] [--email] [--password]`
  - `superbox auth status`
  - `superbox auth refresh`
  - `superbox auth logout`
- `superbox push [--name NAME] [--force]` — security scan + publish to S3 registry.
- `superbox pull --name NAME --client CLIENT` — write client config for Lambda executor.
- `superbox run --name NAME` — interactive JSON-RPC calls to the executor.
- `superbox logs --name NAME [--follow|-f]` — CloudWatch logs with optional tail.
- `superbox search` — list servers from the registry.
- `superbox inspect --name NAME` — open repo URL for a server.
- `superbox test --url URL --client CLIENT [--entrypoint FILE] [--lang LANGUAGE]` — bypass registry/scans for quick trials.

## Examples

```bash
# Register + login (email/password)
superbox auth register
superbox auth login --provider email --email user@example.com

# Google device flow
superbox auth login --provider google

# Publish a server
superbox init
superbox push --name weather-mcp

# Configure for Cursor
superbox pull --name weather-mcp --client cursor

# Test without publishing
superbox test --url https://github.com/user/repo --client chatgpt

# Inspect + logs
superbox inspect --name weather-mcp
superbox logs --name weather-mcp --follow
```

## Notes

- The CLI expects a valid `.env` in the working directory for AWS/Firebase config when pushing/logging.
- Tokens are stored under `~/.superbox/auth.json`.
- For full behavior, see `SuperBox-BE/src/superbox/cli` and `README.md` in the backend repo.
